# Wallet 관리 UI + 상태 흐름 - v0.0.4

## 문제 정의

### 현상
- 인증(World ID) 및 지갑 바인딩(SIWE) 완료 후 사용자가 자신의 지갑 상태를 확인할 수 있는 화면이 없음
- 사용자 여정(인증 → 바인딩 → 확인)의 마지막 단계인 "확인" UI가 부재

### 원인
- Phase 0.0.3까지 인증 및 바인딩 기능 구현에 집중하여, 결과를 보여주는 UI를 구현하지 않음
- 지갑 목록 조회 API가 없음

### 영향
- 사용자가 자신의 지갑이 정상적으로 바인딩되었는지 확인할 수 없음
- 바인딩된 지갑 주소를 복사하거나 활용할 수 없음
- 전체 사용자 여정이 완결되지 않아 UX 흐름이 불완전

### 목표
- 인증/바인딩 완료 후 사용자가 자신의 지갑 상태를 확인할 수 있는 화면 제공
- 전체 사용자 여정(인증 → 바인딩 → 확인)을 화면 흐름으로 완성

## 성공 기준
- [ ] `/wallet` 페이지에서 바인딩된 지갑 목록이 표시됨
- [ ] 각 지갑의 주소(축약), 체인, 인증 상태, 인증 방법이 표시됨
- [ ] 지갑 주소 전체 복사 기능이 동작함
- [ ] 지갑이 없을 때 빈 상태 UI가 표시되고 바인딩 유도함
- [ ] 로딩 중 스켈레톤 UI 표시됨
- [ ] 에러 시 에러 메시지 + 재시도 버튼 표시됨
- [ ] `GET /api/wallet/bindings` API가 세션 기반으로 지갑 목록을 반환함
- [ ] 세션 없는 요청에 401 응답
- [ ] i18n 지원 (ko, en)
- [ ] MiniKit 지갑 주소가 없어도 Wallet Auth 팝업이 정상적으로 표시됨

## 제약사항
- 지갑 삭제/해제는 비범위
- 세금 리포트 생성은 비범위
- `@worldcoin/mini-apps-ui-kit-react` UI Kit 활용
- FE/BE 경계 원칙 준수 (FE는 표시만, BE가 검증/조회 전담)
- i18n은 ko, en만 지원
